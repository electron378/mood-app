How does it work?

The application is made of the 3 key components: a widget that can be injected into almost any web page, a back-end service that handles data collection and a dashboards service that does the data analytics and reporting.

When a person navigates to an intranet page with the widget for the first time, a person will be asked to configure it by selecting the team. That configuration is stored locally in a form of a browser cookie. If configuration is available, the widget will display the vote options and given a mood vote will disappear for a day. The mood vote will be sent to the back-end in an anonymous and pseudo-encrypted form.

The pseudo-encryption mechanism makes an option-vote code time dependent, so a code that meant "happy" today will mean "unhappy" tomorrow, and also what means happy for one user may mean unhappy making the code interception a costly job a modern organization shouldn't allow that anyway.

The back-end will digest the vote-code and append it (as +1) to one of the vote option counters. In this way it is impossible to identify an individual behind the vote by looking at the stored data.
